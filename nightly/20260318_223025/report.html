Category Analysis

Elementwise

Issues: dropout, rope_neox, rope_non_neox, rope_llama31 tests failing. bitwise_not, logical_and, logical_or severely underperforming (ratio=0.33). where, clamp, masked_fill, nan_to_num, alibi, sinusoidal have no test or bench coverage. yarn_rope and longrope not implemented.
Evaluation: The largest and most mature category with solid overall coverage, but the rope family failures and logical op regressions need immediate attention. Fix the rope test failures first as they affect LLM positional encoding correctness, then investigate the 0.33 ratio on bitwise/logical ops which suggests a likely vectorization or dtype issue.

Reduce

Issues: Only 3 of 20 ops implemented (l1_norm, l2_norm, inf_norm), all 3 fail tests. No benchmark data for any op. Core reduce primitives (sum, mean, softmax, argmax, etc.) are entirely unimplemented.
Evaluation: This category is effectively a stub — the 3 implemented ops are broken and the foundational ops (sum, mean, softmax) don't exist yet. Prioritize fixing the linalg_vector_norm test failures since those ops are already implemented, then plan a roadmap for sum/mean/softmax as they are blockers for many downstream categories.

Norm

Evaluation: Healthiest category in the project — 9/10 ops passing with ada_layer_norm variants exceeding baseline. Add test and bench mappings for qk_norm to reach full coverage.

Flash Attention

Issues: mla_decode_fwd (4.33x) and nsa_prefill_fwd (2.43x) significantly exceed baseline. 8 ops unimplemented including flash_prefill_varlen_bwd, flash_decode_varlen_fwd, mla variants, and chunked prefill.
Evaluation: All implemented ops pass tests and the two benchmarked ops show exceptional performance. The missing half of the category covers important production paths (varlen decode, paged MLA, chunked prefill). Prioritize flash_decode_varlen_fwd and flash_chunked_prefill_fwd as they are commonly needed for long-context inference.

MoE

Evaluation: The one implemented op is healthy, but MoE is not usable end-to-end without fused_moe and unpermute_depad. Implement unpermute_depad next as it pairs directly with permute_align, then target fused_moe_deepseek given its industry relevance.

Linear Attention

Evaluation: All tested ops pass functionally, and gated_deltanet_recurrence and gla_recurrence show strong performance. However gated_deltanet_chunkwise at 0.12x is a critical regression that needs root-cause analysis — likely a tiling or memory access pattern issue in the chunkwise kernel. Fix that before expanding to deltanet/retnet.

Issues: Both mamba1 and mamba2 are unimplemented.
Evaluation: No work started. SSM support is important for Mamba-based model families. Begin with mamba2 as it is the more actively deployed architecture, then backport to mamba1.

Overall Assessment

TileOPs has a solid foundation in Elementwise, Norm, and Flash Attention, with 73 of 186 ops fully done and several categories exceeding performance baselines. However, critical LLM inference categories — Reduce, Quantize, Sampling, MoE, and SSM — remain largely unimplemented or broken, limiting end-to-end usability. The project is mid-maturity: the high-performance kernel work is proving out well where it exists, but coverage gaps and a handful of active regressions (rope failures, gated_deltanet_chunkwise, logical ops) need to be resolved before the library can be considered production-ready.

Recommendations

  1. Fix the three active test regressions immediately: rope_neox/rope_non_neox/rope_llama31 failures and the gated_deltanet_chunkwise 0.12x performance regression — these affect core LLM functionality and are likely masking deeper kernel bugs.
  2. Unblock the Reduce category by fixing l1/l2/inf_norm test failures and implementing sum, mean, and softmax, which are transitive dependencies for Norm, Sampling, and attention score computation across the library.
  3. Repair broken benchmark runners for fp8_quantize and chain_speculative_sampling (bench=failed) and ensure all implemented GEMM ops emit ratio values to the log, as missing perf data is preventing accurate health assessment for two high-priority categories.
